# GTG Visual Motif Palette

Status: working motif palette from active reference research  
Date: 2026-06-30  
Related recommendation: `docs/gtg-unique-asset-recommendations.md`

## Purpose

This document defines repeatable visual motifs GTG can own. These are not copied layouts or images. They are small primitives that can become SVGs, CSS treatments, CanvasTexture cards, proposal graphics, and section backgrounds.

## GTG Logo-Derived Geometry

Observed source asset: `public/brand/gtg-logo.png`.

Reusable geometry cues:

- Central vertical spine.
- Three-way fork / Y-like junction.
- Thick diagonal slab segments.
- Cut, angled termini.
- Red active mark against black/charcoal fields.

Use these cues as small accents:

- frame notches;
- active route segments;
- selected topology nodes;
- proof-card corner markers;
- section divider ticks.

Do not repeat the full logo shape as a wallpaper, background pattern, or decorative monogram field.

## Motif 1: Customer Proof Frame

Use for:

- Hero media panels.
- Customer proof grid.
- Proposal proof strips.
- Social preview image.

Visual components:

- Warm-white logo field.
- Charcoal technical frame.
- Fine neutral grid outside the logo field.
- One small red angular marker.
- Optional compact customer name outside the logo safe area.

Rules:

- Never recolor customer marks.
- Never crowd the customer logo.
- Keep the customer name available in HTML.

## Motif 2: Data Lattice

Use for:

- Data & Analytics.
- Capability Map.
- OG image background layer.

Visual components:

- Thin grid.
- Layered rectangular data planes.
- Small red active node.
- Sparse chart topology lines.

Rules:

- Do not imitate dashboards.
- Do not invent metrics.
- Keep labels semantic, not image-only.

## Motif 3: Signal Handoff

Use for:

- Data Streaming.
- Hero-to-section transitions.
- Engagement Flow Map.

Visual components:

- Directional lines.
- Offset event pulses.
- Split/merge paths.
- One red handoff segment.

Rules:

- Use motion only to clarify direction.
- Static version must remain understandable.
- Avoid neon/glow treatments.

## Motif 4: Provisioning Mesh

Use for:

- Infrastructure Automation.
- Platform Golden Path Diagram.
- Technical topology kit.

Visual components:

- Rectangular nodes.
- Structured mesh paths.
- Configuration blocks.
- Red approval/provisioning point.

Rules:

- Do not use cloud-provider icons as GTG icons.
- Keep diagrams abstract and GTG-owned.
- Use adjacent text for meaning.

## Motif 5: Quality Gate Matrix

Use for:

- DevOps & Quality.
- Release confidence visual.
- Test traceability diagram.

Visual components:

- Stage columns.
- Test/result cells.
- Defect/retry loop.
- Red gate marker.

Rules:

- Do not show fake success rates.
- Do not imply DORA metric performance.
- Explain the concept with approved copy.

## Motif 6: Diagnostic Constellation

Use for:

- Consulting & Technical Support.
- Signal Correlation Map.
- Support section illustrations.

Visual components:

- Trace, metric, and log-like paths.
- Dependency nodes.
- Incident focus point.
- Red intervention marker.

Rules:

- Do not imitate observability product UI.
- Keep it as a service-support metaphor.
- No fake alerts or customer incidents.

## Motif 7: Engagement Route

Use for:

- Engagement Flow Map.
- Contact / consultation section.
- Proposal intro slide.

Visual components:

- Five or six route stops.
- Handoff arcs.
- Small document/system nodes.
- Red active path segment.

Rules:

- Requires approved GTG engagement wording.
- Avoid implying guaranteed results.
- Mobile should collapse into a vertical route.

## Rejected Motifs

- Floating glass objects.
- Purple/blue gradient clouds.
- Dense fake dashboards.
- Vendor icon constellations.
- Continuous particle backgrounds.
- Full-screen 3D outside Hero.

## Motion Discipline

Use motion only when it clarifies:

- direction;
- state change;
- handoff;
- active focus;
- section transition.

Do not use motion as the primary proof of credibility. Customer proof, service structure, and capability meaning should remain readable when motion is disabled.

Motion motif priority:

1. Static asset first.
2. Reduced-motion version second.
3. Subtle CSS/GSAP motion third.
4. No new WebGL outside Hero.
